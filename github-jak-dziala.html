Jak działa Git?

Podstawowe pojęcia Git

Git jest systemem kontroli wersji, który śledzi zmiany w plikach w czasie. Działa w sposób rozproszony, co oznacza, że każda kopia repozytorium przechowuje pełną historię projektu, co umożliwia pracę bez stałego dostępu do internetu.

Jak działa Git?

Git działa na zasadzie śledzenia zmian w plikach i przechowywania ich w repozytoriach. Proces ten przebiega w kilku etapach:

  1. Repozytorium: Git przechowuje dane w tzw. repozytoriach. To miejsce, w którym znajdują się wszystkie wersje plików i historia zmian.
  2. Commit: Kiedy wprowadzasz zmiany w plikach, Git umożliwia zapisanie ich w postaci "commitów" – punktów kontrolnych, które przechowują zmiany w projekcie.
  3. Branch (gałąź): Git pozwala na pracę nad różnymi funkcjami w tzw. gałęziach. Każda gałąź jest niezależna i może być rozwijana, a później zintegrowana z główną gałęzią (master/main).
  4. Merge: Kiedy zmiany w różnych gałęziach są gotowe, mogą być łączone za pomocą operacji `merge`.
  5. Push/Pull: Git umożliwia synchronizację zmian pomiędzy lokalnym repozytorium a zdalnym repozytorium (np. na GitHubie) za pomocą komend `git push` i `git pull`.

Git jest niezwykle szybki, efektywny i sprawdza się zarówno w małych, jak i dużych projektach, pozwalając na łatwą współpracę zespołową i zarządzanie wersjami kodu.
